This is a human-action checkpoint. The user must deploy via Vercel Git integration because:
1. Vercel CLI is not installed on the dev machine
2. Git integration is the standard and recommended deployment method for Next.js
3. GEMINI_API_KEY must be configured in the Vercel dashboard (cannot be done via CLI without auth)

**Pre-deployment: Push code to GitHub**
If not already done, push the repository to a GitHub repo.

**Deploy to Vercel (per D-07, D-08):**

1. Go to https://vercel.com/new
2. Click "Import Git Repository" and select your GitHub repo
3. Vercel auto-detects Next.js -- accept default settings:
   - Framework Preset: Next.js
   - Build Command: `npm run build` (auto-detected)
   - Output Directory: `.next` (auto-detected)
4. Before clicking "Deploy", add environment variable:
   - Key: `GEMINI_API_KEY`
   - Value: (your API key from https://aistudio.google.com/apikey)
   - Environment: Production, Preview, Development (all three)
5. Click "Deploy"

**Post-deployment verification:**
1. Visit the Vercel URL (*.vercel.app)
2. Confirm Strategic Overview tab loads
3. Confirm Executive Summary shows AI-generated Korean text (not the old static placeholder)
4. Confirm Market Cards, Threshold Warnings, and other sections load data
5. Wait 1-2 minutes and verify the summary shows (it may take a moment on first load due to Gemini API cold start)

**If Executive Summary shows error state:**
- Check Vercel dashboard -> Deployments -> Function Logs for errors
- Verify GEMINI_API_KEY is set in Vercel Project Settings -> Environment Variables
- Redeploy after fixing
